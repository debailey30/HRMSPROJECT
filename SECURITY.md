# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please follow the steps below to report it:

1. **Do not disclose the vulnerability publicly.** Instead, report it privately to the project maintainers.
2. **Send an email to [security@example.com](mailto:security@example.com)** with the following details:
   - A description of the vulnerability.
   - Steps to reproduce the vulnerability.
   - Any potential impact or severity of the vulnerability.
   - Any suggested mitigation or remediation steps.

The project maintainers will acknowledge the receipt of your report within 48 hours and will work with you to understand and address the vulnerability. We appreciate your efforts to responsibly disclose the vulnerability and will credit you for your contribution if desired.

## Security Updates

We are committed to maintaining the security of this project and will promptly address any reported vulnerabilities. Security updates will be communicated through the project's release notes and changelog.

## Supported Versions

We actively support the following versions of the project:

- Version 1.x
- Version 2.x

If you are using an older version, we recommend upgrading to a supported version to ensure you receive the latest security updates and improvements.

## Security Best Practices

To help maintain the security of this project, we recommend the following best practices:

- Keep your dependencies up to date by regularly checking for and applying updates.
- Use strong, unique passwords for any accounts associated with this project.
- Enable two-factor authentication (2FA) for any accounts that support it.
- Regularly review and audit your project's code and dependencies for potential security issues.
- Follow the principle of least privilege when granting access to your project's resources.

By following these best practices, you can help ensure the security and integrity of this project and its users.
